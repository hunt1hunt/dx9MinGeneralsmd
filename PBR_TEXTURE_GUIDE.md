# PBR 纹理贴图 & 模型制作指南

> 适用于 MinGeneralsmd (dx9MinGeneralsmd) — SAGE 引擎 PBR 管线
> Phase 3 对应文档 — `_pbr.dds` 纹理管线资产制作规范

---

## 一、命名约定

| 文件 | 格式 | 说明 |
|------|------|------|
| `unit_body.tga` | TGA (RGB) | 原有 Albedo 不变，保持原样 |
| `unit_body_pbr.dds` | DXT1/5 | **新文件** — R=粗糙度, G=金属度, B=AO |
| `unit_body_n.dds` | DXT5nm | **新文件** — 切线空间法线贴图 |
| `unit_body_s.dds` | A8 (可选) | 原有高光贴图不变，用于 Legacy 兼容 |

三种文件与 Albedo 纹理**同目录、同 basename**，引擎自动探测加载。

## 二、`_pbr.dds` 通道打包

单张纹理，三个通道分别存不同数据：

| 通道 | 数据 | 值域 | 备注 |
|------|------|------|------|
| **R** | Roughness (粗糙度) | 0.0~1.0 | 0=镜面(玻璃), 1=粗糙(泥土) |
| **G** | Metalness (金属度) | 0.0~1.0 | 0=非金属(dielectric), 1=金属 |
| **B** | AO (环境光遮蔽) | 0.0~1.0 | 0=全遮蔽, 1=无遮蔽 |
| **A** | 未使用 | — | DXT5 的 alpha 留白或复用 roughness |

### ⚠️ 常见通道打包错误

- **不要把 AO 放 alpha** — 渲染时 alpha 通道可能被引擎做特殊处理（透明度测试等），导致 AO 数据丢失
- **不要三个通道都填满 0 或 255** — 极端值会导致渲染全黑（R=0 粗糙度为 0 会变成镜面）或全金属（G=255）
- **不要用 JPEG 压缩** — DDS 格式必须是 DXT1（无 alpha）或 DXT5（保留 AO 精度），优先 DXT5
- **不要反转 Roughness 语义** — Roughness 是"粗糙度"，不是"光滑度"。0=全光滑（玻璃），1=全粗糙（泥土）

## 三、各材质类型的典型参数范围

| 材质 | Roughness | Metalness | AO | 典型对象 |
|------|-----------|-----------|----|----------|
| 金属 (车辆武器) | 0.2~0.6 | 0.8~1.0 | 0.8~1.0 | 坦克履带、炮管、装甲板 |
| 涂漆金属 (车身) | 0.3~0.5 | 0.0~0.2 | 0.8~1.0 | 悍马车体、建筑钢架 |
| 塑料/橡胶 | 0.6~0.9 | 0.0 | 0.7~0.9 | 轮胎、天线、手柄 |
| 混凝土/石材 | 0.7~0.9 | 0.0 | 0.4~0.8 | 建筑墙壁、地面、桥墩 |
| 泥土/沙地 | 0.8~1.0 | 0.0 | 0.5~0.7 | 地表、战壕 |
| 玻璃 | 0.0~0.1 | 0.0 | 0.9~1.0 | 车窗、建筑窗户 |
| 木材 | 0.7~0.9 | 0.0 | 0.6~0.8 | 木箱、栅栏 |

### 💡 重要提醒

目前引擎没有金属度半透明材质的支持。在 Phase 4 (ps_3.0) 实现之前：

- Metalness 建议设置为**极端整数**（0 或 1），不要在 0.5 附近徘徊
- 金属度 0.5~0.9 且粗糙度极低的材质（类似日冕 mod 的"导体"效果）渲染结果不可预测
- 非金属材质的 specular 固定为 4% (0.04)，由 shader 内部硬编码，不需要在贴图中表达

## 四、法线贴图 `_n.dds`

### 技术要求

| 项目 | 要求 |
|------|------|
| 格式 | **DXT5nm** (DXT5 with normal map compression) |
| 颜色空间 | **线性空间**，不要勾选 sRGB |
| 强度 | 建议 0.5~2.0，超过 2.0 会产生渲染闪烁 |
| 导出工具 | 3ds Max / Blender 标准法线贴图插件 |

### 与传统 W3D 法线贴图的区别

- 旧引擎用 `_n.dds` 时走固定管线 DOT3 bump mapping，需要在 alpha 通道存储高度
- PBR 法线贴图走**切线空间**，三通道标准格式，与主流引擎一致
- 如果发现法线方向反了（凹凸变凹陷），需要**翻转绿色通道**（在导出插件中勾选 Flip Y 或手动反转 G 通道）

## 五、UV 映射要求

| 项目 | 原版 W3D 模型 | PBR 优化建议 |
|------|---------------|-------------|
| UV 通道数 | 1 个 | **保留 1 个**，不要创建额外 UV 通道 |
| UV 接缝 | 可见于高光边缘 | **藏在几何转折处**，避免接缝在平面上 |
| 重叠 UV | **允许**（大量使用） | 应**展开避免重叠**，否则 AO 烘焙结果错误 |
| UV 密度 | 不平均 | 尽量均匀，保证 `_pbr.dds` 纹素密度一致 |
| UV 镜像 | 常见 | **必须消除**，镜像 UV 导致法线贴图方向错误 |

### ⚠️ 重叠 UV 是最大的坑

原版 Generals 模型**大量使用重叠 UV** 来省纹理空间（一个 UV  Shell 叠在另一个上面，共享同一块纹理区域）。在 Blinn-Phong 时代这没问题，但 PBR 的 AO（环境光遮蔽）和法线贴图对 UV 重叠**非常敏感**：

- AO 重叠 → 遮蔽值互相污染，接缝处出现暗块
- 法线重叠 → 两个不同朝向的面共用法线数据，出现"凹凸反转"

**建议：在 Blender/Max 中展 UV 时，先检查模型是否有重叠 UV 壳，全部展开后再烘焙 PBR 贴图。**

## 六、模型几何注意事项

| 关注点 | 原版习惯 | PBR 要求 |
|--------|----------|----------|
| 三角面数 | 保留原状 | 不变，旧模型面数对 PBR 足够 |
| 平滑组 | 使用平滑组 | **必须正确设置**，否则法线贴图切线计算错误 |
| 材质分配 | 1 个 W3D 材质含 spec/shininess | **原 W3D 材质参数保留**，供无 `_pbr.dds` 时 Legacy 降级使用 |
| LOD 层级 | 有 LOD 模型 | 所有 LOD 层级推荐同步更新 `_pbr.dds` |
| 顶点色 | 可能使用 | PBR 渲染保留顶点色通道，不要删除 |

### 关于平滑组

PBR 的法线贴图依赖于**正确的切线空间**，而切线的计算依赖平滑组：

- **硬边**（不同平滑组）→ 切线方向突变，法线贴图在该边断开
- **软边**（同平滑组）→ 切线连续，法线贴图过渡自然

如果法线贴图出现"锯齿状接缝"，90% 是平滑组问题。在导出前应检查模型的平滑组分配是否与 UV 接缝一致。

## 七、DDS 导出设置

```
分辨率:      与对应 Albedo 纹理相同（推荐 512² / 1024²）
格式:        DXT5（保留 AO 精度）或 DXT1（无 alpha 场景）
Mip 链:      生成（让 DX9 自动降级）
sRGB:        关闭（粗糙度/金属度/AO 都是线性数据，不应用 gamma）
压缩:        DXT5 对平滑渐变友好，DXT1 块状感更强
```

### 推荐的制作流程

```
Substance Painter / Designer / Photoshop
    ↓ 导出 _pbr.dds (DXT5, sRGB=OFF)
    ↓ 导出 _n.dds (DXT5nm, sRGB=OFF)
    ↓ 原 Albedo (.tga) 保持不变
    ↓ 文件放入 GeneralsMD/Data/Art/Textures/ 对应目录
    ↓ 游戏启动时 Phase 3 代码自动加载
```

### 3ds Max + PS 烘焙工作流（替代方案）

> 参考自 RA3 日冕 mod 的贴图制作流程。不需要 Substance Painter，用 3ds Max 2021+ 的 Simple Map Baker 脚本 + Photoshop 即可快速产出 PBR 纹理。

#### 所需工具

| 工具 | 说明 |
|------|------|
| 3ds Max 2021+ | 需要新版 Bake to Texture API（旧版本不支持） |
| Simple Map Baker | 免费脚本，下载：https://cganimator.com/simplemapbaker/ |
| Photoshop / GIMP | 有任何图层功能的软件即可 |

#### 烘焙步骤

1. **模型准备**
   - 焊接顶点，修复平滑组（建议全选面 → 30° 自动平滑）
   - 活动件分离（炮塔/车体、吊臂、起落架等分开），否则 AO 计算会认为它们直接相接产生不自然的黑块
   - 删除完全不可见的面（底盘底部等），节省贴图面积
   - UV 展开：使用 Max 的自动 UV 切割，不同 UV 块之间至少留 4 像素间隔

2. **运行 Simple Map Baker**
   - 选中模型（可多选，不同材质建议分开 Mesh）
   - 按 X 搜索脚本名打开窗口
   - 勾选 **Ambient Occlusion (AO)** 和 **Curvature Map（曲率图）**
   - 点击 Bake，输出到 `Documents/3ds Max <版本>/sceneassets/images/`

3. **烘焙产物说明**

   | 贴图 | 用途 |
   |------|------|
   | **AO 图** | 环境光遮蔽 — 缝隙、凹陷处暗，暴露处亮 |
   | **曲率图 R 通道** | 凸起棱角 — 边缘磨损、高光加强 |
   | **曲率图 G 通道** | 凹陷沟槽 — 积灰、锈迹、脏渍 |

#### Photoshop 合成流程

```
步骤 1: 底色（迷彩/涂装/固有色）
    ↓ 正片叠底 (Multiply)
步骤 2: AO 图（提供明暗关系和环境遮蔽）
    ↓ 图层模式选"仅变亮"或"相加"
步骤 3: 曲率图 R（凸起棱角 → 边缘磨损发白）
    ↓ 图层模式选"减去"或反色后正片叠底
步骤 4: 曲率图 G（凹陷沟槽 → 凹槽发黑）
    ↓
最终得到 Albedo 漫反射贴图
```

**⚠️ 基于 PBR 理论的勘误（来自日冕作者）：**

- **金属材质在 Albedo 上应该很暗** — 金属几乎不产生漫反射，大多数能量分配给镜面反射。除非表面脏污或锈蚀，否则应按非金属处理
- **如果使用 PBR shader，主贴图不再仅代表漫反射** — 它也会控制金属的反射光谱和菲涅尔效应，不需要在主贴图上增加暗角
- **能量守恒** — SPM 越红（越光滑/金属），漫反射应越暗。我们的 shader 已内建 `diffuse *= (1.0 - 0.3 * metalness)` 补偿

### 日冕 SPM → MinGenerals `_pbr.dds` 转换指南

> 如果你手头有 RA3 日冕 mod 的 PBR 贴图（使用 SPM 贴图承载 PBR 数据的版本），可以用以下公式转换为 MinGenerals 的 `_pbr.dds` 格式。

#### 通道映射

```
日冕 SPM 贴图                          MinGenerals _pbr.dds
┌──────────────┐                     ┌──────────────────┐
│ SPM.R        │ 取反 (1-x)         │ _pbr.R = 粗糙度   │
│ (光滑度编码)  │ ─────────────────→ │                   │
├──────────────┤                     ├──────────────────┤
│ SPM.G        │ 直接拷贝           │ _pbr.G = 金属度   │
│ (独立金属度)  │ ─────────────────→ │                   │
├──────────────┤                     ├──────────────────┤
│ 烘焙 AO 图   │ 直接拷贝           │ _pbr.B = AO       │
│              │ ─────────────────→ │                   │
└──────────────┘                     └──────────────────┘
```

#### 转换公式

```
_pbr.dds.R = 1.0 - SPM.R / 255.0     （光滑度 → 粗糙度，值域取反）
_pbr.dds.G = SPM.G / 255.0            （金属度，直接线性映射）
_pbr.dds.B = AO                         （AO 使用独立烘焙的贴图）
```

#### 日冕 PBR 资产复用检查清单

- [ ] `_pbr.dds` 是否 R=Roughness, G=Metalness, B=AO？ → **直接复制可用**
- [ ] 是否用的是 SPM 编码的 PBR 数据？ → **需用上述公式转换**
- [ ] 法线贴图是 DXT5nm 格式？ → **直接复制可用**
- [ ] 模型是 .w3x 格式（RA3 RNA 引擎）？ → **需在 3ds Max 中重新导出为 .w3d**
- [ ] UV 是否有重叠壳？ → **必须展开消除重叠**
- [ ] UV 通道数 > 1？ → **删除多余 UV 通道，只保留 1 个**
- [ ] 骨骼权重每个顶点 > 1？ → **简化为 1 权重/顶点（SAGE 限制）**

### 推荐的调试方法

1. **检查日志** — `E:\terrain_diag.log` 中的 `PBR_INIT` 行显示注册的 PBR 纹理数量
2. **视觉观察** — 金属度高的区域应有镜面反射高光，粗糙度高的区域应模糊
3. **Legacy 降级验证** — 若无 `_pbr.dds`，自动走旧材质推导，视觉不应比原版差
4. **对比测试** — 同一模型分别放/不放 `_pbr.dds`，确认渲染差异正确

## 八、常见问题排查

### Q: 贴图放好了但看不到效果？
A: 检查文件名是否正确（`_pbr.dds` 全小写），纹理是否在引擎搜索路径内，重启游戏清空纹理缓存。

### Q: 模型看起来全黑？
A: 检查 `_pbr.dds` 的 Roughness 通道。如果 R=0（全黑），表示粗糙度为 0，shader 按镜面处理，没有漫反射。应设 R=128~255（0.5~1.0）。

### Q: 模型看起来像塑料？
A: Metalness 通道可能全是 0（全黑）。金属物体的 G 通道应在 200~255。

### Q: 法线贴图凹凸方向反了？
A: 在导出时翻转绿色通道（Flip G / Invert Y），或在 DDS 工具中手动反转 G 通道。

### Q: UV 接缝处出现暗线？
A: 检查 UV 是否有重叠壳，如有则需要展开。检查平滑组是否与 UV 接缝对齐。
