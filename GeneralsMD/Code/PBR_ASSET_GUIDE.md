## 1. 模型格式：W3D 还是 W3X？

### 结论：使用 **W3D** 格式

| 格式 | 引擎 | 本项目支持情况 |
|------|------|:------------:|
| **W3D** | SAGE（C&C Generals / Zero Hour） | ✅ **原生支持** |
| W3X | RNA（Red Alert 3） | ❌ **不支持** |

**原因：**
- MinGeneralsmd 基于 **SAGE 引擎**（Generals/Zero Hour），原生使用 **W3D** 格式
- **W3X** 是 RA3 的 RNA 引擎格式，虽然基于 W3D 扩展，但骨骼动画系统、材质系统、文件头结构均有差异
- 除非对引擎做大幅修改以支持 W3X 导入，否则 W3X 模型无法被 SAGE 引擎加载

### 建模与导出工具链

```
3DS Max 2021+  →  W3D 导出插件  →  游戏 W3D 文件
      ↓
Simple Map Baker（烘焙曲率/AO）
      ↓
Photoshop（贴图合成）
      ↓
DDS 导出（NVTT/PS 插件） → 游戏纹理
```

> 3DS Max **2023** 及以上版本有社区 W3D 导出工具。旧版本（Max 2010/2012）的官方 W3D 导出插件在新版 Max 上不可用，请使用社区维护版本。

## 2. PBR 管线概述

### 2.1 渲染架构

MinGeneralsmd 使用 **Metallic / Roughness（金属度/粗糙度）** PBR 工作流，采用 **GGX 微表面模型** + **Cook-Torrance BRDF**：

```
Lo(p,ωo) = ∫(f_diff + f_spec) · Li(p,ωi) · (n·ωi) dωi

f_diff = (1-F) · albedo / π               ← 漫反射（Lambert）
f_spec = F · G · D / (4·(n·ωi)(n·ωo))     ← 镜面反射（GGX）

D = GGX(NdotH, α)     ← 法线分布（Trowbridge-Reitz）
G = Smith(NdotL, α)   ← 几何遮蔽
F = FresnelSchlick(VdotH, F0)  ← 菲涅尔
```

### 2.2 SPM 通道映射（次世代 Shader）

本项目的 PBR Shader 对 SPM 贴图的通道定义如下：

```
┌────────────────────────────────────────────────────────────┐
│  R 通道（反射/光滑度+金属度）                              │
│  0–127    (0%–50%)  = 非金属                              │
│  128–191  (50%–75%) = 过渡（锈蚀/覆尘）                    │
│  192–255  (75%–100%) = 金属                                │
│  全域同时控制光滑度连续变化（0=粗糙, 255=光滑）            │
├────────────────────────────────────────────────────────────┤
│  G 通道 = 预留（当前未使用，未来可能改为独立金属度通道）    │
├────────────────────────────────────────────────────────────┤
│  B 通道 = 所属色/阵营色（team color）混色浓度              │
│  0=无阵营色, 255=完全使用阵营色                            │
└────────────────────────────────────────────────────────────┘
```

### 2.3 能量守恒要点

```
SPM越红（反射越强） → 漫反射应该越暗

因为：总反射能量 = 漫反射 + 镜面反射 ≤ 入射能量
金属几乎无漫反射：所有能量分配给镜面反射
非金属有漫反射：镜面反射 F0 固定约 4%（0.04）
```

### 2.4 Shader 实现层次

| 级别 | Shader 模型 | 光照 | IBL | 适用硬件 |
|------|-----------|------|-----|---------|
| 1 | ps_2_0 | 1 太阳光 + 3 额外方向光 | ✗ | SM2.0+ |
| 2 | ps_3_0 | 1 太阳光（循环 GGX） | ✗ | SM3.0+ |
| 3 | ps_3_0 + IBL | 1 太阳光 + 漫反射 IBL | 辐照度 CubeMap | SM3.0+ |
| 4 | ps_3_0 + 完整 IBL | 1 太阳光 + 完整 IBL | 辐照度+预滤波+BRDF LUT | SM3.0+ |

## 4. 建模规范

### 4.1 模型面数与性能

| 类型 | 推荐面数 | 参考 |
|------|:-------:|------|
| 载具（小型） | 2000–4000 | 悍马、技术皮卡 |
| 载具（中型） | 4000–8000 | 帕拉丁、战霸、蝎子 |
| 载具（大型） | 8000–15000 | 帝王坦克、航空母舰 |
| 建筑（小型） | 1000–3000 | 兵营、补给中心 |
| 建筑（大型） | 5000–15000 | 指挥中心、机场 |
| 步兵 | 500–1500 | 单兵 |

### 4.2 棱角处理原则

为了使后续步骤中的边缘曲率计算更显著，**棱角部分应制作得更加锐利**：

- ✅ 减少不必要的切角（Chamfer）或细分（Subdivide）
- ✅ 锐利边缘不添加倒角
- ✅ 风格接近写实或硬朗科幻
- ❌ 不要所有边缘都切角（会导致曲率图太平淡）

### 4.3 平滑组

平滑组对模型观感影响非常大：

```
建议操作：全选所有面 → 30度自动平滑
```

- 30 度以内的相邻面自动平滑过渡
- 超过 30 度的边缘保持锐利
- 手动调整需要锐利的边缘（如装甲接缝）

### 4.4 活动件分离

在预期会转动的部件上，需要将活动件与其他部分做适当分离：

- 炮塔与车体之间
- 吊臂/机械臂
- 车轮（若需要独立旋转）

**目的**：防止 AO 自动计算认为活动件与主体直接相接，导致接缝处出现过黑现象。

### 4.5 预制件策略

将通用配件进行预先建模和贴图绘制，整合到一整个贴图上：

```
通用配件库：
├── 轮子 / 履带
├── 车灯 / 天线
├── 挂钩 / 牵引装置
├── 烟雾弹发射器
├── 光电观瞄
├── 后视镜
└── 排气管
```

优势：
- 简化每个新单位的 UV 展开步骤
- 节约贴图面积
- 形成风格统一的系列感
- 每个模型增加少量 Mesh 是可接受的

### 4.6 不可见面删除

出于性能优化目的，完全看不见的面应当删除：

- 车体底部（若不展示）
- 建筑内部（若不进入）
- 炮塔内部

## 5. UVW 展开与排列

### 5.1 利用 MAX 自动工具

MAX 自带的自动 UV 功能可以一键化操作：

```
自动展开 → 自动切割 → 自动排列
```

比手动排列快得多且更少出错。

### 5.2 UV 分配原则

| 原则 | 说明 |
|------|------|
| **不可见面缩小** | 炮塔底部、车底等看不到的面适当缩小 UV 占比，优化显存利用 |
| **朝向规律** | UV 摆放尽量横平竖直，避免画钢板接缝等条状纹理时出现像素锯齿 |
| **连续面尽量连接** | 增强纹理连续性。UV 块撕裂会导致游戏内平滑组也被撕裂 |
| **分割在锐边** | UV 分割线应放在无需平滑的锐边（平滑组边界或朝下的不可见边缘） |
| **零碎面缩角** | 过于零碎的面自动展开后缩到一个角落即可 |
| **不重叠** | 不同 UV 块不要重叠，边缘保持足够间距 |

### 5.3 UV Padding

UV 编辑器右下角可调整自动排列时的 Padding（边缘填充距离）：

```
建议值：0.01–0.1（取决于预期贴图大小）
最终保证在成品贴图上，任意不同 UV 块之间至少有 4 像素间隔
```

Padding 不足会导致不同 UV 块之间串色（texel bleed）。

### 5.4 对称与镜像

- 任何预期有**不可镜像**内容的部位（文字标识、迷彩图案）不建议粗暴对称切割
- 如有迷彩，避免简单对称分割导致完全对称降低观赏性
- 对称分割可以大幅节省 UV 面积 → 提高有效贴图分辨率

## 6. 自动化烘焙——Simple Map Baker

### 6.1 烘焙概念

烘焙 = 将模型的部分属性依照 UVW 坐标的映射关系，以位图形式绘制到一张给定尺寸的平面图上。

**目的**：通过特定算法模拟环境光照，将 AO 和表面曲率信息直接绘制到位图上，减少手绘工作量，批量增加细节。

### 6.2 操作步骤

1. 选中所需模型（可多选），**不包括贴上去的预制件**
2. 不同材质部分（哑光漆/金属/玻璃）建议分成不同 Mesh，烘焙时自动分图
3. 按 **X 键** 搜索脚本名称唤起 Simple Map Baker 主窗口
4. 勾选需要的烘焙项目：**曲率图（Curvature）** + **环境光遮蔽（AO）**
5. 点击 **Bake** 按钮

### 6.3 烘焙参数建议

| 参数 | 建议值 | 说明 |
|------|:------:|------|
| 贴图尺寸 | 1024² 或 2048² | 与最终输出贴图一致 |
| 采样半径 | 2–4 | 越小曲率越锐利，越大越平滑 |
| 起始阈值 | 0.1–0.3 | 低于此值视为平面，不产生曲率 |
| AO 采样数 | 64–128 | 越高效果越好，烘焙越慢 |
| AO 最大距离 | 10–20 | AO 影响范围 |

### 6.4 烘焙输出

烘焙完成后，位图默认出现在：

```
C:\Users\<用户名>\Documents\3ds Max 2023\sceneassets\images\
```

> 如果更改过"文档"文件夹位置，请在新目录下寻找。

### 6.5 曲率图通道分解

曲率图是后续贴图合成的核心素材：

```
红色通道（R）→ 凸起棱角部分（边缘磨损处）
绿色通道（G）→ 凹陷部分（接缝/刻线）
蓝色通道（B）→ 无信息
黑色区域     → 没有凹凸的平面
```

颜色越亮代表凸起/凹陷的角度越锐利。

### 6.6 进阶烘焙选项

| 选项 | 用途 |
|------|------|
| **世界坐标位置** | 在 PS 中按高度/位置选择区域（如车底尘土、舰船水渍） |
| **世界法线** | 在 PS 中按表面朝向选择区域（如积雪只涂朝上面） |
| **材质 ID** | 按材质种类分区，允许不同材质在交界处模糊融合 |

**世界坐标位置示例：**
- Z 坐标低 → 车辆接近地面的位置 → 贴尘土材质
- Z 坐标高 → 两栖舰船的水线以上 → 防锈漆与水渍

## 7. 漫反射贴图（Diffuse）制作

### 7.1 基础色与 AO 合成

```
操作步骤：

1. 新建 1024×1024（或 512×512）画布
2. 底色：根据阵营确定基本色
   - USA：浅蓝色系
   - China：绿色/土黄色系
   - GLA：土黄/沙色系
   可以是迷彩或其他涂装
3. AO 图层置于顶层，混合模式：正片叠底（Multiply）
   作用：提供基本明暗关系和环境光遮蔽
4. 曲率图的红色（凸起）和绿色（凹陷）通道以灰度形式提取
   - PS：右键提取通道
   - GIMP：颜色→分量→提取通道
```

### 7.2 曲率图层混合

```
图层顺序（从下到上）：
1. 底色图层
2. 曲率凸起图层（红通道灰度）→ 混合模式：仅变亮 / 线性减淡（相加）
3. 曲率凹陷图层（绿通道灰度）→ 混合模式：减去 / 反色后正片叠底
4. AO 图层 → 正片叠底
```

**基于 PBR 理论的注意事项：**

- 金属表面几乎没有漫反射，在漫反射图上应该很暗
- 金属的反射光几乎不受 AO 影响（能量分配给镜面反射）
- 如使用 PBR Shader，主贴图不再仅代表漫反射，也控制金属反射光谱和菲涅尔效应
- 使用 PBR Shader 时不要在漫反射图上额外加暗

### 7.3 进阶技巧：智能材质混合

```
凸起图层 → 设置为 Alpha 蒙版
           → 叠加另一张材质图（锈迹、磨损）
           → 添加噪波图或随机笔刷→ 磨损更随机自然
```

### 7.4 漫反射贴图规范速查

| 材质类型 | sRGB 范围 | 备注 |
|---------|:---------:|------|
| 非金属深色 | 30–80 | 暗部不可低于 30 |
| 非金属亮色 | 180–240 | 纯白（255）仅用于自发光 |
| 金属 | 180–255 | 金属的彩色反射 |
| 纯黑（0,0,0） | — | **禁止使用** |

## 8. SPM 贴图制作

### 8.1 SPM 通道定义

```
SPM 贴图（本项目的 PBR 关键贴图）：
┌──────────────────────────────────────────────────────┐
│ R 通道（反射度+金属度+光滑度）                        │
│   0–42   = 哑光非金属（如涂装、帆布）                │
│   43–127 = 半粗糙非金属（如塑料、木材）              │
│   128–191 = 过渡区（锈蚀金属、污渍金属）             │
│   192–255 = 抛光金属/高反射                          │
│   全域 = 光滑度（数值越高反射越清晰）                 │
├──────────────────────────────────────────────────────┤
│ G 通道 = 预留（未来可能作独立金属度通道）             │
├──────────────────────────────────────────────────────┤
│ B 通道 = 所属色/阵营色浓度                           │
│   0   = 无阵营色                                     │
│   255 = 完全使用阵营色                               │
└──────────────────────────────────────────────────────┘
```

### 8.2 SPM 制作步骤

```
1. 新建与漫反射同尺寸的图层，填充深红色
   - 哑光漆效果：R ≈ 42
   - 半光漆：R ≈ 100–150
   - 抛光金属：R ≈ 200–255

2. 复制曲率图凹凸图层：
   - AO → 正片叠底（Multiply）
   - ⚠️ SPM 里不应该有 AO（镜面反射不受 AO 影响）

3. 曲率凸起图层 → 颜色叠加
   - 示例：边缘漆面磨损 → 露出高反射金属 → 反射度增加
   - 也可以：边缘抛光被磨损 → 反射度减少

4. 手动细节：
   - 玻璃处 → 绘制纯红色（高反射）
   - 漫反射接缝处 → 绘制黑色（不反射）
   - 接缝两侧 → 略亮红色（边缘磨损）
   - 文字标语处 → 降低红色
```

### 8.3 所属色/阵营色处理

```
阵营色也被认为是哑光漆 → SPM B 通道设置:
- 阵营色区域可将反射度适当降低
- B 通道绝不应该有 AO
- 建议将阵营色区域作为单独图层覆盖在 AO 之上

为避免颜色溢出：
- 使用 PBR Shader 时不需要降主贴图亮度（shader 自动修正）
- 阵营色哑光漆受边缘磨损影响 → 使用凸起图层亮度相减
  → 制造阵营色涂层在边缘处变淡的效果
```

### 8.4 能量守恒调整

```
SPM 越红（反射越强） → 漫反射应该越暗

物理原因：
  漫反射能量 + 镜面反射能量 ≤ 入射能量
  高反射表面 = 更多能量分配给镜面反射 = 更少漫反射

只有"极度光滑的浅色绝缘体"（陶瓷、塑料）才能同时有强漫反射和镜面反射
→ 这正是原版贴图"塑料质感"的来源
→ PBR Shader 有自动修正，可忽略此问题
```

### 8.5 SPM 贴图与 PBROverride.ini 的配合关系

#### 引擎的真实处理流程

SPM 贴图和 PBROverride.ini **不是互相冲突的关系**，而是引擎已经设计了明确的优先级处理流程。关键在于 `dx8renderer.cpp` 中的**两阶段流水线**：

```
Phase 3.5（第 1827 行）：设 c3 常数值
  ↓
  从 PBROverride.ini 读取，c3 = { 1.0, rough, 1.0, metal }
  （此时 c3.x=1.0 表示"有 INI 覆盖"）

Phase 4c（第 1848 行）：选择着色器变体
  ↓
  检测该模型是否有 _pbr.dds 贴图？
  ├── 有 → 覆盖 c3 = { 0.0, ..., ..., ... }
  │        （c3.x=0.0 → 禁用常量覆盖 → 使用 SPM 贴图值）
  └── 无 → 保留 Phase 3.5 设置的 c3.x=1.0
           （使用 PBROverride.ini 常数值 → NT 着色器变体）
```

关键代码（`dx8renderer.cpp` 第 1869–1874 行）：

```cpp
// When _pbr.dds exists alongside legacy PBR params, disable constant
// override (c3.x=0) so roughness/metalness/ao come from the texture.
if (hasPBRTex) {
    float pbrParams[4] = { 0.0f, pbrModel->Get_Legacy_Roughness(), 1.0f, pbrModel->Get_Legacy_Metalness() };
    DX8Wrapper::_Get_D3D_Device8()->SetPixelShaderConstantF(3, pbrParams, 1);
}
```

#### 两种路径互不影响

| | 旧模型（无 `_pbr.dds`） | 新模型（有 `_pbr.dds`） |
|---|---|---|
| `Has_Legacy_PBR()` | ✅ true | ✅ true |
| `_pbr.dds` 存在 | ❌ 无 | ✅ 有 |
| `c3.x` 最终值 | **1.0**（INI/模型值覆盖） | **0.0**（贴图控制） |
| roughness 来源 | PBROverride.ini / 模型内嵌值 | **SPM.R** |
| metalness 来源 | PBROverride.ini / 模型内嵌值 | **SPM.G** |
| AO 来源 | 无（=1.0） | **SPM.B** |
| 阵营色来源 | 无 | **SPM.B** |
| 着色器变体 | NT（NoTexture） | 完整 PBR 纹理 |

#### 实际操作

1. **新模型**：做好 SPM 贴图，存为 `xxx_pbr.dds`→ 引擎自动识别并使用它，**无视 PBROverride.ini**。不需要删 INI 条目，不需要改引擎代码。
2. **旧模型**：维持 PBROverride.ini 覆盖，NT 着色器使用常数值。
3. **一个模型不会同时走两条路径**：有 `_pbr.dds` 就走纹理，没有就走 INI。两者互不干扰。

## 9. 法线贴图（NRM）制作

### 9.1 快速方案

新建 **16×16** 文件，填充淡蓝色（R=127, G=127, B=255）即（0.5, 0.5, 1.0）作为平坦法线贴图。

> 这是偷懒方法，但游戏内效果依然整洁。强烈不建议用主贴图粗暴加法线滤镜（效果混乱）。

### 9.2 高质量方案

**方法一：高模烘焙**
```
1. 制作高模（细分 2–3 级）
2. 低模 UV 展开
3. 使用 MAX 的烘培功能将高模法线烘焙到低模
```

**方法二：刻线 + 滤镜**
```
1. 在漫反射贴图上绘制刻线（黑白线条）
2. 使用 NVIDIA Normal Map Filter 或类似工具
3. 从灰度图生成法线贴图
```

### 9.3 法线贴图命名与格式

```
⚠️ 法线贴图命名规则：
   引擎自动检测 <albedo_name>_n.dds 文件
   例：AVPaladin_ALB.DDS → 自动查找 AVPaladin_ALB_n.dds

⚠️ 格式注意事项：

1. 绿通道反转：
   RA3/Generals 使用的不是标准 DirectX 格式
   而是 DirectX 的反转版本（光照从左上角打来）
   如果搬运其他游戏贴图，注意绿通道可能需要翻转

2. 压缩要求：
   DXT 压缩时，蓝通道需全部涂满 255（避免压缩方块）
   RA3 不支持 DXT5NM 格式
   建议格式：RGBA 5551 或 BC5（3Dc）

3. 格式对照：
   DirectX 格式：+X 向右，+Y 向上，+Z 向屏幕外
   OpenGL 格式：+Y 向下（需要翻转 G 通道）
```

## 11. 纹理命名规范

### 11.1 引擎自动检测机制

引擎在加载纹理时，根据 **albedo 贴图的文件名**自动查找关联的 PBR 和法线贴图。

具体代码（`W3DAssetManager.cpp` 第 196–201 行）：

```cpp
// 以 albedo 文件名为基础，追加 _pbr.dds 后缀
pDot = strrchr(pbrName, '.');
if (pDot) {
    *pDot = 0;                       // 去掉原扩展名
    strcat(pbrName, "_pbr.dds");     // 追加 _pbr.dds
    if (TheFileSystem->doesFileExist(pbrName)) {
        registerPBRTexture(filename, pbrName);  // 注册到 PBR 贴图表
    }
}
// 同时探测 _n.dds 法线贴图
strcat(pbrName, "_n.dds");
```

### 11.2 命名规则

```
albedo 文件名决定一切！

例：albedo = "AVPaladin_ALB.DDS"
    → PBR/SPM 贴图：AVPaladin_ALB_pbr.dds  （自动检测）
    → 法线贴图：     AVPaladin_ALB_n.dds     （自动检测）
```

三张贴图的文件名关联关系：

```
AVPaladin_ALB.DDS              ← 漫反射（albedo，模型材质指定）
AVPaladin_ALB_pbr.dds          ← SPM（引擎自动查找）
AVPaladin_ALB_n.dds            ← 法线（引擎自动查找）
```

### 11.3 推荐命名方案

为便于识别，推荐以下命名约定：

| 贴图 | 命名格式 | 示例 |
|------|---------|------|
| Albedo / 漫反射 | `<模型名>_ALB.DDS` | `AVPaladin_ALB.DDS` |
| SPM（PBR Map） | `<模型名>_ALB_pbr.dds` | `AVPaladin_ALB_pbr.dds` |
| Normal / 法线 | `<模型名>_ALB_n.dds` | `AVPaladin_ALB_n.dds` |

> ⚠️ 注意：SPM 和法线贴图的文件名必须基于 **albedo 的完整文件名**（含 `_ALB` 部分），简单写 `<模型名>_pbr.dds` 不会自动匹配。

## 13. PBROverride.ini 配置

### 13.1 文件格式

```ini
; 注释以 ; 或 // 开头
<mesh_prefix> = <roughness>, <metalness>  ; 可选注释

示例：
  AVPaladin     = 0.40, 0.45   ; 帕拉丁坦克
  AVHUMMER      = 0.55, 0.25   ; 悍马
```

### 13.2 匹配规则

- **前缀匹配**：`AVPaladin` 匹配 `AVPaladin.Body01`、`AVPaladin.Turret01`
- **最长匹配优先**：`AVPaladin_D` 优先于 `AVPaladin` 匹配 `AVPaladin_D.Body01`
- **下一个字符必须是 `.` 或 `\0`**：`AVPaladin` **不**匹配 `AVPaladin_D.Body01`
- 大小写不敏感（`_strnicmp`）

### 13.3 变体命名

| 容器名 | 含义 | 需要 INI 条目 |
|--------|------|:------------:|
| `AVPaladin` | 完整状态 | ✅ 基础条目 |
| `AVPaladin_D` | 轻损状态 | ✅ |
| `AVPaladin_D1`~`_D4` | 中重度损伤 | ✅（与 _D 同值） |
| `AVPaladin_H` | 残骸 | ✅（可选） |
| `AVPaladin_Laser` | 激光阵营变体 | ✅（可选） |
| `AVPaladin_SW` | 超级武器阵营变体 | ✅（可选） |

> ⚠️ 以上变体条目**仅对无 `_pbr.dds` 的旧模型有效**。新模型有 `_pbr.dds` 贴图时，INI 直接被 Phase 4c 覆盖为 `c3.x=0`，这些变体条目不会生效。

## 15. 常见问题

### Q1: 没有 PBR 贴图怎么办？

引擎自动退化到 **NoTexture 模式**，使用 PBROverride.ini 配置值或模型内嵌的 `Legacy_PBR()` 值。NoTexture 模式下粗糙度/金属度在模型上均匀。

### Q2: 金属和非金属交界处有白边？

M/R 流程固有问题。缓解方法：
- 交界区域使用渐变过渡
- 确保 UV texel 密度足够
- PBR 贴图分辨率不要过低

### Q3: 怎么让金属看起来更亮？

增加 SPM R 通道值（提高金属度 + 降低粗糙度），同时确保：
- IBL CubeMap 亮度足够
- 环境光包含对金属的贡献
- 光源方向与观察方向形成高光角

### Q4: 这是 Substance Painter 还是 3DS Max 工作流？

**本指南使用 3DS Max + Simple Map Baker + Photoshop 工作流**，不使用 Substance Painter。详见第 3–6 章。

### Q5: SPM 贴图命名必须是 `xxx_pbr.dds` 吗？

**是的，必须带后缀 `_pbr.dds`**。引擎根据 **albedo 贴图的完整文件名**（含 `_ALB` 部分）自动拼接 `_pbr.dds` 来查找关联的 PBR 纹理。详见第 11 章。

### Q6: 支持哪些纹理格式？

引擎加载 DDS（DXT1/BC1、DXT3/BC2、DXT5/BC3、BC4/ATI1、BC5/ATI2、BC7）和 TGA（32-bit RGBA）。推荐 DDS。
