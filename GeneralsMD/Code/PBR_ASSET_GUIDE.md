## 1. PBR 管线概述

### 1.1 渲染架构

MinGeneralsmd 使用 **Metallic / Roughness（金属度/粗糙度）** PBR 工作流，采用 **GGX 微表面模型** + **Cook-Torrance BRDF**：

```
Lo(p,ωo) = ∫(f_diff + f_spec) * Li(p,ωi) * (n·ωi) dωi

f_diff = (1-F) * albedo / π           ← 漫反射（Lambert）
f_spec = F * G * D / (4 * (n·ωi)(n·ωo))  ← 镜面反射（GGX）

F = Fresnel-Schlick（Schlick近似）
G = Smith-GGX（几何遮蔽）
D = GGX-Trowbridge-Reitz（法线分布）
```

### 1.2 Shader 实现层次

| 级别 | Shader 模型 | 光照 | IBL | 适用硬件 |
|------|-----------|------|-----|---------|
| 1 | ps_2_0 | 1 太阳光 + 3 额外方向光 | ✗ | 旧 GPU（SM2.0+） |
| 2 | ps_3_0 | 1 太阳光（循环 GGX） | ✗ | SM3.0+ |
| 3 | ps_3_0 + IBL | 1 太阳光 + 漫反射 IBL | 辐照度 CubeMap | SM3.0+ |
| 4 | ps_3_0 + 完整 IBL | 1 太阳光 + 完整 IBL | 辐照度 + 预滤波 + BRDF LUT | SM3.0+ |

均支持 NoTexture（NT）模式（不使用 PBR 纹理贴图，由模型或 INI 提供常数值）。

### 1.3 寄存器布局

```
s0 = Albedo/Diffuse 纹理
s1 = Normal 纹理（预留）
s2 = PBR 纹理（R=粗糙度, G=金属度, B=AO）
s3 = 辐照度 CubeMap（IBL）
s4 = 预滤波环境 CubeMap（IBL specular）
s5 = BRDF LUT 纹理（IBL）

c0 = 太阳方向（归一化）
c1 = 太阳颜色
c2 = 相机位置（世界空间）
c3 = { 1.0, roughness, 1.0, metalness }   ← override 参数
c4-c9 = 额外方向光 x3
c10 = 环境光颜色
c11 = 调试模式标志
```

## 2. 纹理贴图规范

### 2.1 贴图清单

| 贴图 | 通道 | 色彩空间 | 格式 | 分辨率建议 | 备注 |
|------|------|---------|------|-----------|------|
| **Albedo** | RGB | sRGB | DXT1/BC1 | 1024² ~ 4096² | 不含光照信息 |
| **PBR Map** | RGB | 线性 | DXT5/BC3 | 512² ~ 2048² | R=粗糙度, G=金属度, B=AO |
| **Normal** | RGB | 线性 | DXT5/BC3 | 1024² ~ 4096² | DirectX 格式（+X 右） |
| **Specular** | Alpha | 线性 | DXT5/BC3 | — | 仅旧版 Specular 流程用 |

> **PBR Map** 是本项目的关键贴图 —— 它将粗糙度、金属度、AO 三通道合并到一张纹理中，节省显存和采样次数。

### 2.2 各贴图详细规范

#### Albedo（基础色）

| 材质类型 | sRGB 范围 | 参考值 | 备注 |
|---------|----------|--------|------|
| 非金属深色 | 30–80 | 50 | 暗部不可低于 30 |
| 非金属亮色 | 180–240 | 220 | 纯白（255）仅用于自发光 |
| 金属 | 180–255 | 220 | 金属的彩色反射 |
| 纯黑（0,0,0） | — | — | **禁止使用** —— 会导致错误的金属反射 |

**禁止：**
- ❌ 包含光照/阴影信息（烘焙光照会破坏 PBR 能量守恒）
- ❌ 纯黑区域（用于 Albedo）
- ❌ 金属区域使用饱和度过低的值

**允许：**
- ✅ 微观 AO（接触阴影）
- ✅ 颜色变化（锈迹、污渍、涂装磨损）

#### PBR Map（R=粗糙度, G=金属度, B=AO）

最关键的自定义贴图，三通道编码：

```
┌─────────────────────────────────────────────────┐
│   R通道（粗糙度）  │   G通道（金属度） │   B通道（AO）    │
├────────────────────┼──────────────────┼─────────────────┤
│   0.0 = 光滑       │   0.0 = 非金属    │   0.0 = 全遮蔽   │
│   1.0 = 粗糙       │   1.0 = 金属      │   1.0 = 无遮蔽   │
│   sRGB 编码         │   线性            │   线性           │
└────────────────────┴──────────────────┴─────────────────┘
```

**PBR Map 制作流程：**

1. 在 Substance Painter 中完成材质
2. 导出时重映射通道：
   - Red ← Roughness
   - Green ← Metallic
   - Blue ← AO（环境光遮蔽）
3. 导出为 TGA 或 DDS（DXT5/BC3）
4. 引擎加载时以线性色彩空间采样

> 对于不支持的着色器版本，引擎会自动退化到 NoTexture 模式，使用模型/INI 提供的常数值。

参考数值：

| 材质 | R（粗糙度） | G（金属度） | B（AO） |
|------|:----------:|:----------:|:-------:|
| 坦克涂装装甲 | 0.40 | 0.45 | 0.80 |
| 悍马漆面 | 0.55 | 0.25 | 0.75 |
| 战机蒙皮 | 0.55 | 0.12 | 0.85 |
| 舰船涂装钢质 | 0.45 | 0.35 | 0.80 |
| 帆布 | 0.70 | 0.00 | 0.70 |
| 铸铁（裸露） | 0.60 | 0.85 | 0.75 |
| 锈蚀金属 | 0.70 | 0.40 | 0.70 |

#### Normal Map（法线贴图）

- 格式：**DirectX**（+X 向右，+Y 向上，+Z 向屏幕外）
- **不要**使用 OpenGL 格式（会反转 Y 通道，导致光照错误）
- 强度：建议 1.0–3.0，过大导致"纸片"感

### 2.3 纹理命名规范

```
<模型名>_<贴图类型>.<扩展名>

示例：
  AVPaladin_ALB.DDS       // Albedo
  AVPaladin_NRM.DDS       // Normal
  AVPaladin_PBR.DDS       // PBR Map (R=rough, G=metal, B=AO)
  AVPaladin_SPEC.DDS      // Specular (旧版，可选)
```

## 3. 模型规范

### 3.1 网格命名

W3D 模型使用 **容器.部件** 命名结构，前缀匹配系统根据容器名称查找 PBR 参数：

```
AVPaladin.Body01       ← 容器 "AVPaladin" → 匹配 PBROverride.ini 中的 "AVPaladin"
AVPaladin_D.Body01     ← 容器 "AVPaladin_D" → 需要单独条目 "AVPaladin_D"
AVPaladin.Turret01     ← 同样匹配 "AVPaladin"
```

**规则：**
- 容器名称（`.` 之前的部分）用于 PBROverride.ini 匹配
- 容器名 + `_D`、`_D1`、`_D2` 表示损伤态（需要各自条目）
- 容器名 + `_H` 表示残骸
- 前缀匹配要求下一个字符为 `.` 或 `\0`，所以 `AVPaladin` **不会**匹配 `AVPaladin_D`

### 3.2 模型层级（适用于 Legacy PBR）

对于没有 PBR 纹理的模型，引擎支持在 W3D 模型文件中嵌入 roughness/metalness：

```
MeshModelClass::Set_Legacy_PBR(roughness, metalness)
```

这需要在模型导出时设置，通常通过 W3D 工具链的材质属性接口完成。

### 3.3 UV 要求

- UV 通道 0（TEXCOORD0）：用于 Albedo 采样
- UV 通道 1（TEXCOORD1）：预留（当前未使用）
- Texel 密度：建议 **至少 512²/单位面积**，防止远距离闪烁
- UV 岛之间留 2–4 像素 padding，防止 mipmap 色溢

### 3.4 LOD 层级

| LOD | 三角面比例 | PBR 贴图分辨率 |
|-----|:---------:|:-------------:|
| LOD0 | 100% | 原分辨率 |
| LOD1 | 60–70% | 1/2 |
| LOD2 | 30–40% | 1/4 |
| LOD3 | 10–15% | 1/8（或 NT 模式） |

> LOD1+ 建议保留 PBR Map（可降分辨率），极端 LOD 可退化到 NoTexture 模式。

## 5. PBROverride.ini 配置

### 5.1 文件格式

```
; 注释以 ; 或 // 开头
<mesh_prefix> = <roughness>, <metalness>  // 可选注释

示例：
  AVPaladin     = 0.40, 0.45   ; 帕拉丁坦克
  AVHUMMER      = 0.55, 0.25   ; 悍马
```

### 5.2 匹配规则

- **前缀匹配**：`AVPaladin` 匹配 `AVPaladin.Body01`、`AVPaladin.Turret01` 等
- **最长匹配优先**：`AVPaladin_D` 优先于 `AVPaladin` 匹配 `AVPaladin_D.Body01`
- **下一个字符必须为 `.` 或 `\0`**：`AVPaladin` **不**匹配 `AVPaladin_D.Body01`
- 大小写不敏感（`_strnicmp`）

### 5.3 各变体命名对照

| 容器名 | 含义 | 需要 INI 条目 |
|--------|------|:------------:|
| `AVPaladin` | 完整状态 | ✅（基础条目） |
| `AVPaladin_D` | 轻损状态 | ✅ |
| `AVPaladin_D1`~`_D4` | 中重度损伤 | ✅（一般与 _D 同值） |
| `AVPaladin_H` | 残骸 | ✅（可选） |
| `AVPaladin_SW` | 超级武器阵营变体 | ✅（阵营特有涂装可选） |
| `AVPaladin_Laser` | 激光阵营变体 | ✅（同上） |

## 6. 导出与压缩建议

### 6.1 纹理压缩

| 贴图 | 推荐格式 | 备选 | 说明 |
|------|---------|------|------|
| Albedo | BC1（DXT1） | BC7（高质） | 无 Alpha 时用 BC1 |
| PBR Map | BC3（DXT5） | BC7 | 需要保留三通道信息 |
| Normal | BC5（3Dc） | BC3（DXT5） | BC5 两通道压缩质量更高 |
| AO（独立） | BC4（ATI1） | — | 单通道 |

### 6.2 分辨率策略

```
主模型（近距离）：
  Albedo:  2048²
  PBR Map: 1024²     ← 粗糙度/金属度不需要极高的分辨率
  Normal:  2048²

中型模型（中距离/载具）：
  Albedo:  1024²
  PBR Map: 512²
  Normal:  1024²

小型/远距离（建筑/步兵）：
  Albedo:  512²
  PBR Map: 256²
  Normal:  512²
```

### 6.3 色彩空间与编码

```
┌──────────────┬──────────┬──────────┐
│    贴图      │ 游戏内   │ DDS 编码 │
├──────────────┼──────────┼──────────┤
│ Albedo       │ sRGB     │ sRGB     │
│ PBR Map      │ 线性     │ 线性     │
│ Normal       │ 线性     │ 线性     │
│ AO           │ 线性     │ 线性     │
└──────────────┴──────────┴──────────┘
```

> **⚠️ 重要**：PBR Map 必须以**线性**色彩空间编码。如果以 sRGB 编码，粗糙度/金属度值会被 Gamma 校正，导致渲染结果产生肉眼可见的色偏。

## 7. 诊断与调试

### 7.1 诊断日志

设置 `PBR_DEBUG_MODE` 环境变量或修改 `GlobalData.m_pbrDebugMode` 可启用逐像素调试：

| 调试模式 | 效果 |
|:-------:|------|
| 0 | 正常渲染 |
| 1 | 仅显示粗糙度（R 通道可视化） |
| 2 | 仅显示金属度（G 通道可视化） |
| 3 | 仅显示 AO（B 通道可视化） |
| 4 | 仅显示法线（NDotL 可视化） |
| 5 | 仅显示漫反射 IBL |

### 7.2 运行时诊断

游戏运行时，`terrain_diag.log` 文件记录了每个 PBR 着色器的编译状态：

```
[timestamp] terrain_pbr = 0           ← S_OK
[timestamp] pbr_unit_ps30 = 0         ← S_OK
[timestamp] sun_glow_compiled = 1     ← 编译成功

PBR_UNIT_DIAG #1: shader=0x12345678 nt=1 hasTex=0
  c0(sunDir)=(0.809,-0.379,0.449)
  c1(sunColor)=(1.000,1.000,1.000)
  c10(ambient)=(0.220,0.204,0.172)
```

### 7.3 常见调试流程

1. **黑模** → 检查 PBR Map 是否有有效纹理（非全黑）、着色器编译是否成功
2. **过亮/过曝** → 检查太阳颜色和环境光强度是否过大（不宜 > 3.0）
3. **金属感太强** → 降低 metalness（G 通道）
4. **太暗/无高光** → 降低 roughness（R 通道）
5. **异常高光** → 检查法线贴图格式（DirectX ≠ OpenGL）
6. **闪烁** → 检查 mipmap 链是否完整、texel 密度是否足够

## 8. 常见问题

### Q1: 模型没有 PBR 贴图怎么办？

引擎自动退化到 **NoTexture 模式**，使用模型文件中的 `Legacy_PBR(roughness, metalness)` 值，或 PBROverride.ini 中配置的值。NoTexture 模式下，粗糙度/金属度在整个模型上均匀。

### Q2: 金属和非金属交界处有白边？

这是 M/R 流程的固有问题。在引擎层面已通过 GGX 的 Fresnel-Schlick 近似做了缓解。贴图制作时应：
- 在交界区域使用渐变过渡
- 确保 UV texel 密度足够
- PBR Map 分辨率不要过低

### Q3: 为什么我的 Albedo 看起来比原版更暗？

请检查：
- Albedo 是否以 sRGB 编码（非线性）
- 环境光（c10）强度是否合适（0.2–0.4 为典型范围）
- 太阳颜色（c1）是否在 0.8–1.5 范围内
- PBR 能量守恒本身会减少漫反射亮度，这是正确的物理行为

### Q4: 怎么让金属看起来更亮？

增加金属度（PBR Map G 通道），同时降低粗糙度（R 通道）。金属的反射光来自环境，所以确保：
- IBL CubeMap 亮度足够
- 环境光（c10）包含对金属的贡献
- 光源方向与观察方向形成高光角

### Q5: 支持哪些纹理格式？

引擎加载 DDS（DXT1/BC1、DXT3/BC2、DXT5/BC3、ATI1/BC4、ATI2/BC5、BC7）和 TGA（32-bit RGBA）格式。推荐使用 DDS 以获得 mipmap 链和 GPU 压缩。
